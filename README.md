# 骰评 · 私人电影思想地图

> Every Film is a Landscape of Ideas.

🎲 **骰评**是一台电影思想地图探索引擎。输入电影名，系统首先生成电影的**入口地图**（结构类型识别 + 递进式入口节点），然后在拉康、齐泽克、福柯、马克思等理论家的视角下，从命题空间中采样一条分析路径，输出一个有观点密度的电影解读。

它不是影评生成器。它是**观点采样引擎**：同一部电影可以掷出截然不同的批判视角——有时从「马克思×阶级×规训」切入，有时从「拉康×欲望×移置」展开。每掷一次，就是在电影的思想地图上标记一条新的探索路径。

## 四层架构

```
电影
  ↓
入口地图（结构类型识别 + 递进入口节点）    ← 层 1：电影的结构 DNA
  ↓
命题空间（框架限定 + 骰子加权采样）        ← 层 2：从结构场生成命题
  ↓
观点路径（骰子驱动命题选取）              ← 层 3：骰子采样单条路径
  ↓
骰评（论证 + 反证 + 理论 + 追问）         ← 层 4：输出
```

## 核心特性

### 入口地图（Entry Map）

每部电影先被识别为一种**结构宇宙**（人格结构/空间结构/时间结构/权力结构/欲望结构/意识结构……），再生成递进式入口节点。不同类型的电影获得不同结构的入口地图。

- 《搏击俱乐部》→ 人格裂变：主体→镜像→消费人格→暴力人格→毁灭人格
- 《寄生虫》→ 空间结构：半地下室→豪宅→楼梯→地下室→暴雨
- 《一一》→ 欲望结构：后脑勺照片→东京街道→咖啡馆等待→寺庙哭泣→葬礼独白

### 两层命题筛选

- **第一层**：骰子绕出框架（如「拉康」），生成以拉康理论视角限定的 8 条命题
- **第二层**：骰子的其余维度（问题、对象、机制、崩塌方向）加权筛选，从 8 条中选取最优匹配

### 双轨阅读系统

- **Inside**：进入电影文本内部，分析空间/叙事/镜头语法
- **Around**：围绕电影的外部阅读，分析工业逻辑/观众合约/平台算法
- 自动混合模式，也可手动切换

### 55 位导演数据库

系统内置 55 位全球核心导演的结构化风格档案。AI 提取电影时自动识别导演，将导演的 theory_affinity 注入骰子加权，用导演签名词汇覆盖导演骰面。

### 三引擎架构

- **AI 引擎**（DeepSeek V4 / OpenAI）：完整管道，入口地图 + 命题空间 + 观点采样
- **本地引擎**（Ollama）：支持 hermes3:8b / qwen3 等本地模型
- **模板引擎**：零成本即时出结果，无需 API

## 快速开始

```
打开 → 输入电影名 → 掷骰 → 保存卡片
```

模板引擎无需任何配置。AI 引擎需要 API Key（⚙ → 选 provider → 填 Key）。

## 本地模型

```bash
ollama pull hermes3:8b
# 打开骰评 → ⚙ → Ollama（本地）→ 选模型 → 无需 API Key
```

## 本地运行

```bash
cd release
python -m http.server 8000
# http://localhost:8000
```

或双击 `release/start_server.bat`。

## 部署

GitHub Pages / Netlify / 任何静态托管——将 `index.html` + `directors.js` 放在同一目录即可。
